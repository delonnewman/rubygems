Gem Specification Format

Note: Bold properties are required.

Required Specification Attributes

spec.name [String] the name of your Gem
spec.version [String] the version of your Gem
spec.platform [String] the platform name (see Gems::Platform module for list
spec.date [Time] the date/time that the gem was created
spec.summary [String] short description of Gem library
spec.require_paths [Array] list of [String] require paths from root of gem path

Specifying Files

spec.files [Array] list of file names (as strings)

 example:
 spec.files = Dir.glob("lib/**/*").delete_if {|item| item.include?("CVS")}
 # selects all files/directories in the lib subdirectory excluding CVS files.

Optional Specification Attributes

spec.autorequire [String] the Ruby file to require when the Gem is loaded
spec.author [String] name of the author
spec.email [String] email address of the author
spec.homepage [String] URL of the project/author
spec.description [String] detailed description of the Gem library

Specifying Dependencies

A dependency specifies a Gem (and optional version requirement) that needs to be present for the defined Gem to operate. For example, if an RDF parser used REXML to parse XML files, the corresponding RDF Gem could specify the Gem dependency (on the REXML Gem). When a Gem is required (with require_gem) all dependent Gems are loaded with require_gem. In the aforementioned example the REXML Gem would automatically be loaded if you did require_gem on the RDF Gem. The format for added a dependency is:

spec.add_dependency "name", "version requirement"

Version and Version Requirement Format

A version can be any sequence of number separated by periods. Versions are specified as strings.

 example:
 "0.3"
 "1.34.23"

A version requirement is a version string prefaced with a operator. If an operator is not specified in a version requirement string, it assumes an equal "=" operator. The operator can be one of the following:

 =  Equals version
 != Not equal to version
 >  Greater than version
 <  Less than version
 >= Greater than or equal to
 <= Less than or equal to
 example:
 "> 1.1"
 "= 4.56.4"
